Base hw

  • git config --global user.name "Your Name" - задає ім'я глобально (достатньо виконати один раз).
  • git config --global user.email your_email - задає емейл глобально (достатньо виконати один раз).
  • git init - ініціалізація git-репозиторія.
  • git status - відображає стан робочого каталогу (є чи немає що комітити).
  • git add - добавляє файли робочого каталогу в буфер для подальшого коміту.
  • git commit - добавляє вміст буфера в репозиторій.
  • git log - відображає усі коміти.
  • git checkout - перехід на іншу гілку.
  • git branch - перегляд наявних гілок, або створення нової.
  • git push - відправлення файлів на віддалений репозиторій.
  • git pull - завантажує зміни з віддаленого репозиторію в локальний каталог.
  • git merge - злиття двох гілок.
  • git clone - створення клону віддаленого репозиторію.

Що таке .gitignore

Файл .gitignore потрібен для приховування файлів і папок від системи контролю версій Git.

Advanced hw

  • git revert - створюється новий коміт, який відміняє останній коміт. Обидва коміти надалі існують в історії комітів.
  • git reset - відміняє останній коміт, при цьому стирає його з історії.
  • git rebase - перебазовує робочоу гілку на іншу гілку, тобто змінює основу робочої гілки.
  • git cherry-pick - переносить певний коміт з одного місця в репозиторії в інше.
  • git stash - зберігає зміни внесені до файлу в окремому сховищі на локальному репозиторії і робить відкат до початкової робочої версії файлу.
